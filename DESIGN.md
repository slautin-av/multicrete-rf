## Концепция дизайна сайта

**«Стальной Vega + розовый неон» на ТЁМНОЙ приборной панели.** Реализация бренда в вебе: сайт выглядит как **тёмный приборный пульт** — матовые стальные панели на почти чёрном фоне, утопленные шкалы характеристик, кнопки-клавиши с физическим объёмом, розовое неоновое свечение на главных действиях. Поверх всего — **живой интерактивный «дым»** (WebGL fluid), реагирующий на курсор: фон «дышит», усиливая ощущение работающего прибора.

**Цель:** клиент-инженер с первого взгляда понимает «это не маркетинговый лендинг, это инструмент». Тёмная тема + тактильный скевоморфизм + дым отличают сайт от плоских светлых конкурентов в нише промышленных покрытий.

> **Важно про тему:** сайт **тёмный** (светлый текст на тёмных панелях). Ранняя версия DESIGN.md (v1.x) описывала светлую тему — она устарела, тема эволюционировала в тёмную на этапах вёрстки. Все значения ниже — фактические.

## Дизайн-токены (`:root`) — эталон

Единый набор переменных, идентичный в `landing.html` и `calculator/calculator.css`. **Всегда обращаться через `var(--…)`, не хардкодить hex.**

```css
:root {
  /* Действие / информация */
  --blue:    #2F66A5;   /* синий металлик — информация */
  --blue-d:  #244F80;   /* тёмный синий (грани, ховеры) */
  --pink:    #EB4D8C;   /* розовый неон — ДЕЙСТВИЕ (только CTA) */
  --pink-d:  #A52350;   /* нижняя грань клавиши-кнопки */

  /* Тёмная тема — поверхности и текст */
  --ink:     #EAF0F7;   /* основной светлый текст */
  --steel:   #212C3D;   /* панель / карточка на тёмном фоне */
  --scale:   #19222F;   /* утопленные шкалы — темнее панели */
  --line:    #34435A;   /* контуры / разделители */
  --muted:   #93A1B5;   /* второстепенный текст */
  --bg:      #1A2332;   /* базовый тёмный фон (фолбэк за дымом) */

  /* Типы материалов (кодирование) */
  --concrete:#6B7280;   /* Бетон */
  --metal:   #4A6580;   /* Металл */
  --chem:    #2F66A5;   /* Химзащита */
  --hydro:   #1E5973;   /* Гидро */

  --red:     #D64550;   /* критические ошибки — единственное исключение из палитры */
  --max:     1200px;    /* максимальная ширина контента */

  --f-head:  'Oswald', sans-serif;   /* заголовки */
  --f-body:  'Manrope', sans-serif;  /* основной текст */
  --f-tech:  'Tektur', sans-serif;   /* метки, теги, «технические» подписи */
}
```

**Акцент в тексте на тёмном фоне:** чистый `--blue #2F66A5` теряет контраст на тёмном, поэтому для **текстовых** акцентов (метки секций, цифры-статы, `.accent` в заголовках) используется осветлённый синий **`#7FB0E6`**. Заливка кнопок и фон-плашки остаются на `--blue`/`--pink`.

**Шрифты** подключаются через Google Fonts с `display: swap`:
`Oswald` (500/600/700) · `Manrope` (400/500/600/700) · `Tektur` (500/600).

## Фон: интерактивный WebGL-дым (фирменная фишка)

Главный визуальный сигнатурный приём сайта. **Раньше DESIGN.md запрещал WebGL — теперь это осознанное ядро дизайна.**

```css
html { background: var(--bg); }   /* тёмный фолбэк, если холст не поднялся */
body { background: transparent; } /* контент поверх дыма */

#fluid-bg {
  position: fixed; inset: 0; width: 100%; height: 100%;
  z-index: -1;            /* ПОЗАДИ всего контента */
  pointer-events: none;   /* холст НИКОГДА не перехватывает клики */
  display: block;
}
```

- Библиотека: `assets/js/webgl-fluid.umd.js` (локальный файл, не CDN), холст `#fluid-bg`.
- Координаты курсора пробрасываются в дым скриптом (сам холст `pointer-events:none`, поэтому кнопки/ссылки работают поверх).
- Hero-секция делается прозрачной (`.hero-dark`), чтобы дым сильнее всего проступал на первом экране.
- **Фолбэк обязателен:** тёмный `--bg` на `<html>` — если WebGL недоступен, сайт остаётся читаемым тёмным.

**Что по-прежнему НЕ используем:** параллакс-эффекты с фоновыми картинками, автоматические карусели/слайдеры, 3D-повороты на скролле, преувеличенные анимации (scale > 1.2, резкие blur). Дым — единственный «тяжёлый» эффект, и он декоративный и ненавязчивый.

## Скевоморфные техники в коде

Всё — на **CSS** (тени, фаски, внутренние тени, градиенты). Значения ниже — фактические, на тёмных поверхностях.

### Физический объём карточки (`.card`)

```css
.card {
  background: var(--steel);        /* #212C3D — тёмная панель */
  border-radius: 12px;
  padding: 1.5rem;
  box-shadow:
    0 8px 24px rgba(26,35,50,.08),
    0 2px 4px rgba(26,35,50,.04),
    inset 0 1px 0 rgba(255,255,255,.6);  /* верхний блик-фаска «металла» */
  transition: transform .2s, box-shadow .2s;
}
.card:hover { transform: translateY(-2px); /* + тень плотнее */ }
```

### Утопленная шкала характеристики (`.spec-cell`)

```css
.spec-cell {
  background: var(--scale);        /* #19222F — темнее панели */
  border-radius: 8px; padding: 1rem 1.25rem;
  box-shadow:
    inset 0 2px 4px rgba(26,35,50,.08),   /* «утопленность» */
    inset 0 -1px 0 rgba(255,255,255,.8);  /* подсвеченная нижняя грань */
}
.spec-num { font-family: var(--f-head); font-size: 1.8rem; color: #7FB0E6; } /* акцент на тёмном */
```

### Кнопка-клавиша с нижней гранью (главная CTA — розовый неон)

```css
.btn-primary {
  background: var(--pink); color: #fff; border-radius: 10px;
  box-shadow:
    0 4px 0 var(--pink-d),               /* тёмная нижняя грань — клавиша */
    0 0 24px rgba(235,77,140,.3);        /* мягкое розовое свечение */
  transition: transform .1s, box-shadow .1s;
}
.btn-primary:hover  { transform: translateY(1px); box-shadow: 0 3px 0 var(--pink-d), 0 0 32px rgba(235,77,140,.45); }
.btn-primary:active { transform: translateY(3px); box-shadow: 0 1px 0 var(--pink-d), 0 0 16px rgba(235,77,140,.4); }
```

Эффект: клавиша физически продавливается и усиливает свечение.

### Тонкая светящаяся розовая линия снизу (баннеры/hero/оборудование)

```css
.hero-orb::after, .hero-banner::after, .equip-media::after {
  content: ''; position: absolute; left: 16px; right: 16px; bottom: 0; height: 2px;
  background: linear-gradient(90deg, transparent, rgba(235,77,140,.6), transparent);
  filter: blur(1px);
}
```

Эффект: «лампа приборной панели» подсвечивает медиа-блок снизу.

## Компоненты сайта

### Реально собрано на лендинге (одностраничник)

Сайт сейчас — **один длинный лендинг** `landing.html` (не каталог). Секции сверху вниз: шапка-навигация (тёмное стекло, sticky) → hero (баннер + плашка-лента + блок «до/после» с зумом картинок + статы + CTA) → проблема → технология → сферы → применение → ROI → сравнение (таблица) → ТТХ (табы) → доверие/отзывы → **блок «С чего начать»** (калькулятор + скачать опросной лист) → форма заявки → оборудование М700 → контакты → подвал. Плюс два модальных окна: калькулятор и заявка.

Ключевые компоненты и их классы:
- **Секция:** `.section` (padding 80px), `.wrap` (max 1200px), `.sec-label` (Tektur, `#7FB0E6`), `.sec-title` (Oswald, clamp 1.9–2.5rem), `.sec-sub` (muted).
- **Шапка `.header`:** sticky, тёмное стекло `rgba(20,28,40,.72)` + `backdrop-filter: blur(10px)`. Логотип с белым абрисом (drop-shadow), чтобы читался на тёмном.
- **Гексагон `.hex`:** `clip-path` шестиугольника. Для **функциональных иконок** clip-path снимается — остаётся контурная иконка с градиентной обводкой (см. «Иконки»). Залитый гекс используется точечно (`.strip-hex` в hero — синий).
- **Утопленные шкалы `.spec-cell` / `.spec-num`:** цифры характеристик как окна на панели.
- **Таблицы** (сравнение `.cmp-table`, ТТХ `.specs-table` с табами `.specs-tab`): строки на `--steel`/`--scale`, шапка на `--blue`.
- **Форма заявки `.form-box`:** тёмная панель; инпуты `.pf-input` — утопленные (`--scale`, inset-тень), фокус — синий контур; чекбокс согласия 152-ФЗ; буллеты-перки с розовой светящейся точкой.
- **Модалка `.modal`:** тёмная `#212C3D`, оверлей `rgba(26,35,50,.5)` + blur. Калькулятор и заявка — в модалках.
- **Блок «до/после» (`.dp__*`):** пара картинок с зумом ×1.2 по hover/тапу, затенение неактивной, розовый разделитель по центру, бейджи «до» (контур) / «после» (розовый неон).

### План на будущее (ещё НЕ реализовано — каталог материалов)

Карточка материала каталога, сетка каталога 2–3 в ряд, гексагон-стикеры типа задачи с пиктограммой, микро-индикатор статуса «Новинка/Бестселлер». Это описано как **целевой** компонент для будущего каталога; на текущем лендинге его нет. При постройке каталога — держать ту же тёмную систему и токены.

### Бейдж типа задачи (кодирование цветом)

Гексагон/метка с цветом типа: Бетон `--concrete #6B7280`, Металл `--metal #4A6580`, Химзащита `--chem #2F66A5`, Гидро `--hydro #1E5973`.

## Анимация и интерактив

| Эффект | Где | Как |
|---|---|---|
| **Живой дым** | Фон всего сайта | WebGL fluid, реагирует на курсор; `z-index:-1`, `pointer-events:none` |
| **Hover карточки** | `.card` | Подъём `translateY(-2px)` + тень плотнее |
| **Hover/Click кнопки** | `.btn-primary` | Свечение усиливается; при клике «продавливается» `translateY(3px)` |
| **Зум до/после** | `.dp__frame img` | `scale(1.2)` по hover (десктоп) или тапу (`.is-zoomed`, JS); неактивная картинка затеняется |
| **Свечение-линия** | под баннерами/hero/оборудованием | Розовый градиент с `blur` |
| **Розовая точка-индикатор** | буллеты перков, статусы | Мягкое розовое свечение |

**`prefers-reduced-motion`** — переходы у зум-картинок отключаются. **Не используем:** параллакс, автокарусели, 3D-повороты, scale > 1.2.

## Иконки

**Библиотека:** [Lucide](https://lucide.dev) — тонкие линии 1.5–2px, инженерная геометрия. Подключается CDN-скриптом, рисуется через `data-lucide="…"`.

### Фирменный приём: контурные иконки с градиентом синий→розовый

Функциональные иконки в карточках секций — **контур** (без фона) с **градиентом по обводке: синий `#2F66A5` → розовый `#EB4D8C`** по диагонали.

Реализация: один скрытый `<svg>` с `<linearGradient id="brand-grad">` в начале `<body>`; на иконках — `stroke: url(#brand-grad); stroke-width: 1.6`. В CSS у гекс-контейнеров функциональных иконок снимается `clip-path`/фон, остаётся только градиентная обводка.

Чистый розовый как **заливка** остаётся только на CTA — здесь розовый входит лишь «искрой» на краю иконки (декоративная подпись, не функциональный сигнал).

## Edge cases (граничные состояния)

| Состояние | Что показывать |
|---|---|
| **Ошибка валидации формы** | Сообщение под полем, цвет `--red #D64550` (единственное исключение из палитры) |
| **Форма успешно отправлена** | «Спасибо, свяжемся в течение часа» + CTA |
| **Пустой результат/каталог** | «Ничего не найдено» + сброс фильтра + предложить заявку |
| **Загрузка** | Скелет-заглушки на `--scale`, без спиннеров |
| **404** | «Страница не найдена» + ссылки на разделы |

## Доступность и производительность

### Доступность
- Тёмная тема: основной текст `--ink #EAF0F7` на панелях `--steel/--scale` — высокий контраст. Для **текстовых** акцентов на тёмном — осветлённый синий `#7FB0E6` (не тусклый `--blue`).
- Холст дыма `pointer-events:none` — не мешает фокусу и кликам; вся интерактивность доступна с клавиатуры (Tab).
- `prefers-reduced-motion` учитывается; фолбэк `--bg` при недоступном WebGL.
- Цвет — не единственный носитель смысла (иконки + подписи).

### Производительность
- **WebGL-дым — единственный тяжёлый эффект.** Холст один, `webgl-fluid.umd.js` локальный. На слабых устройствах допустимо снижать интенсивность/отключать (фолбэк — тёмный фон).
- Шрифты — Google Fonts CDN, `display: swap`.
- Скевоморфизм — чистый CSS, браузер не грузит.
- Изображения — оптимизировать (WebP + lazy-loading).
