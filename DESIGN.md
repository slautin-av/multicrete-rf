## Концепция дизайна сайта

**«Стальной Vega + розовый неон»** — реализация бренда МультиКрит в вебе. Каждая страница и компонент — как **раздел приборного пульта**: матовый стальной корпус, утопленные шкалы характеристик, кнопки-клавиши с физическим объёмом, гексагон-стикеры с тенью, розовое неоновое свечение на главных действиях и индикаторах.

**Цель:** клиент-инженер с первого взгляда понимает «это не маркетинговая лендинг-страница, это инструмент». Скевоморфные техники работают на ощущение **тактильности и инженерности**, отличая сайт от плоских конкурентов в той же нише.

## Скевоморфные техники в коде

Все техники реализуются **CSS** (тени, фаски, внутренние тени, градиенты). SVG используется только для иконок-пиктограмм. **3D-эффекты, JS-анимации перспективы, WebGL — не используются.**

### Физический объём карточки

```css
.card {
  background: #E8EBEF;
  border-radius: 12px;
  box-shadow:
    /* Мягкая падающая тень снизу */
    0 8px 24px rgba(26, 35, 50, 0.08),
    0 2px 4px rgba(26, 35, 50, 0.04),
    /* Светлая фаска-блик сверху (имитация металлического блика) */
    inset 0 1px 0 rgba(255, 255, 255, 0.6);
}
```

Эффект: карточка «лежит» на странице как физический объект.

### Утопленные шкалы характеристик (внутренняя тень)

```css
.spec-cell {
  background: #F4F6FA;
  border-radius: 8px;
  box-shadow:
    /* Внутренняя тень сверху — эффект «утопленности» */
    inset 0 2px 4px rgba(26, 35, 50, 0.08),
    /* Лёгкая светлая обводка снизу — «подсвеченная шкала» */
    inset 0 -1px 0 rgba(255, 255, 255, 0.8);
}
```

Эффект: цифры характеристик читаются как окна на приборной панели.

### Кнопка-клавиша с нижней гранью (розовый неон для главной CTA)

```css
.btn-primary {
  background: #EB4D8C;
  color: #ffffff;
  border-radius: 10px;
  /* Тёмная нижняя грань — «клавиша нажимается» */
  box-shadow:
    0 4px 0 rgba(165, 35, 80, 1),    /* нижняя грань тёмного оттенка розового */
    0 0 24px rgba(235, 77, 140, 0.3); /* мягкое розовое свечение вокруг */
  transition: transform 0.1s, box-shadow 0.1s;
}

.btn-primary:hover {
  transform: translateY(1px);
  box-shadow:
    0 3px 0 rgba(165, 35, 80, 1),
    0 0 32px rgba(235, 77, 140, 0.45);
}

.btn-primary:active {
  transform: translateY(3px);
  box-shadow:
    0 1px 0 rgba(165, 35, 80, 1),
    0 0 16px rgba(235, 77, 140, 0.4);
}
```

Эффект: клавиша физически продавливается при нажатии и слегка усиливает розовое свечение.

### Гексагон-стикер

SVG с `clip-path: polygon(...)` или встроенный SVG, с тенью:

```css
.hex-badge {
  /* шестиугольник через clip-path */
  clip-path: polygon(50% 0%, 100% 25%, 100% 75%, 50% 100%, 0% 75%, 0% 25%);
  background: var(--badge-color);   /* зависит от типа задачи */
  /* Тень — стикер приклеен поверх карточки */
  filter: drop-shadow(0 2px 6px rgba(26, 35, 50, 0.15));
}
```

Эффект: гексагон выглядит как наклейка-стикер на корпусе.

### Тонкая светящаяся линия снизу (только для главной карточки-героя)

```css
.card-hero::after {
  content: '';
  position: absolute;
  left: 16px;
  right: 16px;
  bottom: -2px;
  height: 2px;
  background: linear-gradient(
    90deg,
    transparent,
    rgba(235, 77, 140, 0.5),
    transparent
  );
  filter: blur(2px);
}
```

Эффект: лампа приборной панели подсвечивает герой-карточку снизу.

## Компоненты сайта (главные)

### Карточка материала (для каталога)

См. `_research/design-refs/claude-design-итерация-3-финальная.html` — готовая HTML-реализация. Структура:

- Верх (~40% высоты): фото объекта (или плейсхолдер с молекулярным паттерном)
- Гексагон-стикер с пиктограммой типа на стыке фото и текста
- Название серии (Oswald, 20–24 px)
- Описание задачи (Manrope, 14–16 px, 1–2 строки)
- Сетка характеристик 2×2 (утопленные шкалы)
- Кнопка-клавиша «Подробнее» (розовый неон)
- Опциональный микро-индикатор статуса в углу (розовая точка для «Новинка» / «Бестселлер» / «Акция»)

### Бейдж типа задачи

Гексагон с иконкой-пиктограммой:
- **Бетон** `#6B7280` — иконка куба
- **Металл** `#4A6580` — иконка гайки/шестерни
- **Химзащита** `#2F66A5` — иконка колбы с каплей
- **Гидро** `#1E5973` — иконка капли или волны

### Главная CTA-кнопка

Розовый неон `#EB4D8C` с нижней гранью и мягким свечением. См. CSS выше.

### Вторичная CTA (для edge case «нет в наличии»)

Контурная розовая кнопка (тот же `#EB4D8C`), но как обводка 2 px вместо заполнения. Текст — розовый. Сохраняет узнаваемость цвета действия в любом состоянии.

### Микро-индикатор статуса

Кружок 8–10 px розового неона с мягким свечением вокруг. Размещается в правом верхнем углу карточки. Внутри — короткая метка («Новинка», «Бестселлер», «Акция»), либо просто как точка-сигнал.

### Утопленная шкала характеристики

Прямоугольная ячейка с внутренней тенью сверху и светлой обводкой снизу. Внутри — крупная цифра (Manrope 600, табличные цифры) и подпись (Manrope 400, uppercase 11 px).

## Адаптивность

Брейкпойнты:

| Брейкпойнт | Ширина | Что меняется |
|---|---|---|
| Mobile | < 640 px | Карточки в 1 столбец, отступы 16 px, шрифты на 15–20% мельче |
| Tablet | 640–1024 px | Карточки в 2 столбца, отступы 24 px |
| Desktop | > 1024 px | Карточки в 3 столбца, отступы 32 px, максимум 1200 px ширины |

Скевоморфные техники сохраняются на всех брейкпойнтах. На мобильном — слегка снижаем интенсивность теней (для производительности), но **не отключаем**.

## Анимация и интерактив

| Эффект | Где | Как |
|---|---|---|
| **Hover карточки** | Карточка каталога | Лёгкий подъём (`transform: translateY(-2px)`) + усиление тени (`box-shadow` плотнее на 20%) |
| **Hover кнопки-клавиши** | CTA | Усиление розового свечения (см. CSS выше) |
| **Click кнопки-клавиши** | CTA | Кнопка «продавливается» (`transform: translateY(3px)`) |
| **Появление карточек при скролле** | Каталог | Fade-in + лёгкий slide-up, длительность 300 мс, easing `cubic-bezier(0.4, 0, 0.2, 1)` |
| **Подсветка шкалы при наведении** | Утопленные ячейки | Очень лёгкая подсветка фона `#F4F6FA → #FFFFFF` с переходом 200 мс |
| **Розовая лампочка-индикатор** | Микро-индикатор статуса | Лёгкое пульсирование яркости свечения (subtle, не агрессивное) |

**Никаких:**
- 3D-перспектив на скролле
- Параллакс-эффектов с фоновыми картинками
- Преувеличенных анимаций (scale > 1.05, поворотов, blur-эффектов)
- Автоматических каруселей и слайдеров

## Иконки

**Библиотека:** [Lucide](https://lucide.dev) — открытая, бесплатная, инженерная стилистика (тонкие линии 1.5–2 px, геометрические формы).

### Фирменный приём: контурные иконки с градиентом синий→розовый (решение 2026-06-12)

Функциональные иконки в карточках секций (проблема, технология, сферы, применение, контакты) рисуются **контуром** (без заливки фона) с **градиентом по обводке: синий металлик `#2F66A5` → розовый неон `#EB4D8C`** (по диагонали, x1=0 y1=0 → x2=100% y2=100%).

Реализация: один скрытый `<svg>` с `<linearGradient id="brand-grad">` в начале `<body>`, и `stroke: url(#brand-grad)` на иконках Lucide (`stroke-width: 1.6`).

**Почему допустимо смешение цветов здесь** (в обход общего правила «синий=информация, розовый=действие, не смешиваем»): в иконке преобладает синий, розовый входит лишь как «искра» на краю — это декоративная фирменная подпись, а не функциональный сигнал. **Чистый розовый как заполненный цвет действия остаётся только на CTA-кнопках** — там разделение функций сохраняется. Найдено на уроке 3.06 переносом приёма из презентаций NotebookLM, отфильтровано в бренд-цвета (бирюзу из презентаций НЕ берём — запрещена в brand.md).

Использование:
- В карточках секций — контурные градиентные (фирменный приём выше)
- В пунктах меню навигации
- В формах (рядом с полями ввода)
- В кнопках действий (рядом с текстом)

## Edge cases (граничные состояния, требуют отдельной разработки)

| Состояние | Что показывать |
|---|---|
| **Пустой каталог под фильтром** | «Под ваш запрос ничего не найдено» + кнопка «Сбросить фильтр» + предложение оставить заявку |
| **Ошибка валидации формы заявки** | Красное сообщение под полем «Это поле обязательно» (красный — единственное исключение из палитры, для критических ошибок). |
| **Состояние загрузки страницы материала** | Скелет-заглушка с серыми блоками (`#E8EBEF`), без спиннеров |
| **Форма успешно отправлена** | «Спасибо, мы свяжемся с вами в течение часа» + основная CTA «На главную» |
| **404 — страницы не существует** | «Страница не найдена» + ссылка на каталог + ссылка на «Подобрать материал» |
| **Карточка материала «нет в наличии»** | Угловая лента «НЕТ В НАЛИЧИИ» + приглушение карточки + вторичная CTA «Узнать о поступлении» (розовый контур) |

## Производительность и доступность

### Производительность

- Все логотипы и фотографии оптимизированы (WebP с фолбэком на PNG, lazy-loading).
- Шрифты грузим через Google Fonts CDN с `display: swap`.
- Скевоморфные техники реализованы CSS — не нагружают браузер.
- Целевая скорость загрузки главной — < 2.5 с на 4G, < 1 с на десктопе.

### Доступность (ARIA, контраст, навигация)

- Контраст текста и фона — минимум 4.5:1 (для основного текста), 3:1 (для крупных заголовков).
- Розовый `#EB4D8C` на белом фоне — контраст 3.1:1. **Для розового текста на белом — НЕ используем**, только для кнопок с белым текстом (там контраст 4.8:1).
- Все интерактивные элементы доступны с клавиатуры (Tab navigation).
- ARIA-роли для всех нестандартных элементов (гексагон-бейджи как `role="img"` с aria-label).
- Поддержка `prefers-reduced-motion` — отключаем анимации появления для пользователей с ограничениями.
