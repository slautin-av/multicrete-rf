# Стартовый промпт для Claude Design — карточка материала

Готовый промпт для первого захода в Claude Design по сценарию А (wireframe → визуал).
Объект: универсальная карточка серии материала в каталоге `multicrete.ru/materials/`.

Workflow: копируешь промпт ниже → claude.ai/design → New project → прикрепляешь референсы → вставляешь промпт → Create Wireframe.

После результата делаешь скриншот → возвращаешься в Claude Code в этом проекте → продолжаешь цикл «дирижёр + руки».

## Что нужно приложить ПЕРЕД промптом

1. **URL для Web Capture:** `https://www.international-pc.com` — главный референс по структуре карточек продуктов.

3. **Скриншот логотипа МультиКрит** (горизонтальный или шар) — для понимания фирменного знака.

## Сам промпт (копировать всё ниже от линии до линии в чат Claude Design)

---

Сделай wireframe карточки материала для каталога B2B-сайта промышленных полимерных покрытий «МультиКрит» (multicrete.ru).

**Что это за элемент:**
Карточка одной серии материала в каталоге. Каталог состоит из 8 таких карточек, расположенных сеткой по 2-3 в ряд. Каждая карточка — компактный анонс материала со ссылкой на полную страницу.

**Аудитория:** инженеры и снабженцы строительных компаний, которые подбирают материал под конкретную задачу (защита бетона / металла / гидроизоляция / химзащита). Им нужна быстрая, информативная карточка без маркетингового шума.

**Что должно быть в карточке:**

- Фото объекта с нанесённым покрытием (заглушка-плейсхолдер для wireframe)
- Шестигранный бейдж (гексагон) с типом задачи: «Бетон» / «Металл» / «Гидро» / «Химзащита» — отсылка к бензольным кольцам полимеров
- Название серии: «МК-160 ЭЛАСТОМЕР», «МК-310 ХИМЗАЩИТА» и т.п.
- Короткое описание задачи (1-2 строки): «Эластомерное покрытие для динамических нагрузок на бетоне»
- 3-4 ключевые характеристики цифрами (адгезия / температурный диапазон / срок службы / толщина слоя)
- Кнопка «Подробнее» — ведёт на полную страницу материала

**Структура карточки (иерархия сверху вниз):**

1. Фото объекта (верх карточки, ~40% высоты)
2. Гексагон-бейдж задачи (на пересечении фото и текстовой части, как наклейка)
3. Название серии (главный акцент типографики)
4. Описание задачи (1-2 строки)
5. Сетка характеристик (4 ячейки 2×2 или одной строкой)
6. Кнопка «Подробнее» (внизу, чёткая CTA)

**Стиль:**

Возьми за основу:
- Структуру карточек продуктов с приложенного референса international-pc.com (через Web Capture)
- Скевоморфный неоморфизм в духе ретро-аудиотехники Vega 109 / Романтика (приложенный скриншот) — мягкие физические объёмы, чувство «физического объекта на столе», а не «цифровой плашки»

**Шрифты:**
- Заголовки и название серии — Oswald
- Описание и характеристики — Jost

**Цветовая палитра** (применяй на этапе визуала, не wireframe):
- Основной фон карточки: тёплый светло-серый (#E8E8E8 — #F2F2F2)
- Глубокие тени и контраст: антрацит (#212121)
- Акцент один: насыщенный (детали и кнопка)
- Гексагон-бейдж: контрастный, но без бирюзы

**Чего НЕ делать:**
- Без бирюзовых и тиловых акцентов
- Без шрифтов с засечками (никакого Playfair-стиля)
- Без сильно скруглённых кнопок-таблеток — углы 8–12 пикселей максимум
- Без мигающих точек статуса
- Без иконок в кружочках
- Без светло-серых градиентов «в духе AI 2024»
- Без «тёмного hero на светлом фоне» как стандарта

**Нестандартные приёмы:**
- Гексагон-бейдж как ритмический элемент (намёк на полимеры)
- Скевоморфные тени — карточка «лежит» на странице как физический объект
- Большая цифра ключевой характеристики (например — толщина слоя) как акцент

**Состояния, которые тоже нарисуй (edge cases):**

- Карточка с отсутствующим фото — что показывать вместо изображения?
- Карточка с длинным названием серии в две строки — как ведёт себя сетка?
- Карточка «временно нет в наличии» — визуальное состояние

**Сейчас нужен только wireframe** — чёрно-белая структура. К цветам и шрифтам перейдём вторым промптом, после того как структура утверждена.

## После того как Claude Design нарисовал

1. Делаешь скриншот результата.
2. Возвращаешься в Claude Code в проекте `multicrete-ru`.
3. Пишешь: «Вот что вышло, прикладываю скриншот. Что просить следующим — переход к визуалу или ещё уточнить структуру?»
4. Claude Code анализирует и решает дальнейший шаг (большой промпт / точечный комментарий / ползунки).

## Куда подключить DESIGN.md (когда он будет)

Когда `DESIGN.md` и `multicrete-brand/brand.md` будут собраны (между уроком 3.05 и 3.06) — в промпт сверху добавится одна строка:

> «Опирайся на DESIGN.md этого проекта и brand.md из multicrete-brand. Цвета и шрифты бери оттуда, я их не повторяю.»

Это сократит промпт раза в два — основные ориентиры будут жить в файлах, а промпт будет только про конкретный элемент.
