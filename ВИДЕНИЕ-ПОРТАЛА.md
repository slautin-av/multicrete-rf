## Функция 1. Калькулятор покрытия по СЛОЖНОСТИ детали

**Главный принцип: считает не площадь, а сложность детали.** Лист металла метр-на-метр и импеллер/подающий винт — разная вселенная трудозатрат.

Что влияет на трудозатраты (оси расчёта):
- **Площадь** покрытия — геометрия.
- **Сложность детали** (главный драйвер по деньгам): нужна ли оснастка/приспособа; нужно ли вращение для равномерного слоя; есть ли незакрываемые зоны (маскировка); доступность поверхности.
- **Материал / технология / толщина** — материальная часть.
- → **Итог: предварительная стоимость + гарантия.**

**Сердце калькулятора = формализованный метод расчёта Алексея.** Сейчас он считает по своей калькуляции (Excel + инженерное чутьё), нюансы заводит руками. Задача — вытащить это в **явную модель**: классы сложности детали с коэффициентами и правила («вращаемое тело → +оснастка +вращение», «незакрываемые зоны → +маскировка»). Машина не придумывает цену — она **прогоняет правила Алексея**. Это та же работа, что со сборкой базы знаний «Хауса» (`multicrete-lab`): тацитный метод → явные коэффициенты, как технологическая карта.

> ⚠️ **ЗАГОТОВКА от Алексея (нужна до стройки):** вытащить из головы/Excel классы сложности и коэффициенты. Без них движок пустой.

**Два режима:**
- **Заказчику** — упрощённый, «предварительный»: выбрал тип детали + примерный размер → получил **вилку/ориентир** стоимости.
- **Алексею** — полный: доводит площади и сложность вручную → **точная цифра для КП**.

> 🔒 **House-фильтр (бизнес-правило):** публично НЕ выкладывать полную модель ценообразования. Коэффициенты сложности — это know-how и маржа. Публичный калькулятор даёт только **вилку**, точный движок остаётся у Алексея. Иначе конкурент за вечер вытащит себестоимость.

## Функция 2. Автокейсы (портфолио работ как инструмент)

**Архитектура — разделить «писателя» и «читателя»:**
- **Писатель = IDE-ассистент** (Claude Code, как «Хаус») — генерит кейсы. **НЕ живёт на сайте.**
- **Хранилище кейсов** — структурные записи (поля + фото). Источник истины.
- **Читатель = сайт** — показывает кейсы из хранилища.

**Как ассистент делает кейс:** Алексей даёт фото (до/после) + описание (можно голосом): деталь, проблема/износ, принятое решение (покрытие, толщина, технология), стоимость, гарантия, результат. Ассистент собирает из этого **структурный кейс** → кладёт в хранилище → сайт его показывает.

**Что видит заказчик на сайте:** листает каталог кейсов, находит **похожую на свою** деталь, кликает → видит, что предстоит сделать, по какой технологии, какой толщины, примерную стоимость, гарантию и результат.

> «Автоматически на сайт» — это спектр. **v1:** ассистент сделал структурный кейс → он лёг в данные → сайт отрендерил все кейсы. Полный авто-публиш «скинул фото в дороге → само висит на сайте» — **поздний слой**, не v1, иначе закопаемся в инфраструктуре раньше, чем будет что показывать.

> Возможная синергия: ассистент кейсов может опираться на знания из `multicrete-lab` (технологии, рецептуры, стоимости уже там) — не делать дважды.

## Как примерно нарежется на этапы (уточнить в discuss-phase)

1. **Калькулятор-движок** — формула площадь × сложность × материал/толщина → стоимость + гарантия (логика, без красоты).
2. **Калькулятор на сайте** — форма для заказчика (режим «вилка») + полный режим Алексея.
3. **Хранилище кейсов** — структура записи (поля + фото).
4. **Ассистент кейсов** — IDE-инструмент: фото+описание → структурный кейс → в хранилище.
5. **Каталог кейсов на сайте** — просмотр, поиск похожей детали.
