# Ручной чек-лист приёмки — Этап 2 (окно калькулятора, форма, встраивание)

Интерфейс и доставку почты автотестами не покрыть. Этот чек-лист — ручная приёмка
этапа. Каждый пункт: **действие → ожидаемый результат**. Отмечайте `[x]` по факту.

**Где проверять:** открыть `landing.html` (точка входа клиента) и/или
`calculator/index.html` (модуль автономно). Для проверок мобильной вёрстки —
DevTools device mode (F12 → значок устройства) или реальный телефон.

---

## 0. Авто-проверка движка (база перед ручной приёмкой)

- [ ] **Движок расчёта зелёный** → в терминале выполнить:
      `node calculator/engine.selftest.js`
      → вывод PASS (расчёт сверен с эталоном «Калькуляция МультиКрит 700»).
- [ ] **Чистая логика валидации зелёная** (опционально) →
      `node --test calculator/calculator-ui.test.js` → все тесты pass.

## 1. Интерфейс калькулятора (UI)

### UI-01 — Окно открывается и закрывается
- [ ] На лендинге нажать заметную кнопку **«Получить расчёт»** (в первом экране, hero)
      → поверх страницы всплывает окно калькулятора, лендинг под ним затемнён.
- [ ] Нажать крестик (×) в углу окна → окно закрывается, клиент остаётся на лендинге.
- [ ] Открыть снова → клик по тёмному фону вне окна → окно закрывается.
- [ ] Открыть снова → нажать клавишу **Esc** → окно закрывается.

### UI-02 — Поля ввода
- [ ] В окне есть поля: **площадь детали (м²)**, **толщина покрытия (мм)**,
      выбор **класса сложности 1/2/3**, галочка **«Деталь тяжелее 80 кг»**.
- [ ] Отметить галочку **«>80 кг»** после расчёта → меняется только **текст оговорки**
      про массу (появляется предупреждение о грузоподъёмном оборудовании),
      **итоговое число НЕ меняется** (CALC-07).

### UI-03 — «Шапка» сметы
- [ ] Ввести площадь/толщину/класс → «Рассчитать стоимость» → показана «шапка»:
      **трудозатраты, вспомогательные материалы, накладные расходы,
      плановые накопления, НДС, транспорт = 0, командировки = 0, материалы,
      ИТОГО и стоимость за ₽/м²**.

### UI-04 — Расшифровка классов и оговорки видны сразу
- [ ] В окне (ещё до расчёта) видна **расшифровка классов 1/2/3** (примеры деталей и коэффициенты).
- [ ] После расчёта виден **блок оговорок** (1 день, 1 специалист, без транспорта/командировок/оснастки;
      грузоподъёмное только при детали >80 кг).

### UI-05 — Валидация ввода
- [ ] Оставить площадь = 0 (или пусто) → «Рассчитать» → подсказка
      **«Укажите площадь больше нуля, м²»**, расчёт не выполняется и не ломается.
- [ ] Толщина = 0 → подсказка **«Укажите толщину больше нуля, мм»**.
- [ ] Класс не выбран → подсказка **«Выберите класс сложности»**.
- [ ] После исправления полей повторный расчёт проходит нормально.

### UI-06 — Фирменный стиль и работа на телефоне
- [ ] Окно в **тёмной фирменной теме** сайта (та же палитра, что у лендинга).
- [ ] На телефоне (DevTools device mode или реальный телефон): окно открывается во весь экран,
      **внутри окна можно прокрутить** все поля и результат, **фон страницы не дёргается**
      при прокрутке окна, кнопки/поля нажимаются пальцем.

## 2. Встраивание в сайт (INT)

### INT-01 — Модуль на чистом стеке
- [ ] Калькулятор живёт в папке `calculator/` (по аналогии с `materials/`),

      (а не прокручивает к блоку цен).
- [ ] Существующие формы лендинга и окно **«Заказать звонок»** (кнопка «Звонок инженера»)
      продолжают работать как раньше — открываются, отправляются, не сломаны.

## 3. Заявка / обратная связь (LEAD)

### LEAD-01 — Цена без формы, форма необязательна
- [ ] После расчёта **ИТОГО и «шапка» видны сразу**, без заполнения формы заявки.
- [ ] Форма заявки (имя/организация/телефон/e-mail) расположена **ниже** результата
      и заполнять её необязательно для просмотра цены.

### LEAD-03 — Согласие 152-ФЗ
- [ ] Заполнить имя, **не отмечая** галочку согласия → «Отправить заявку» →
      браузер блокирует отправку и подсвечивает обязательное согласие.
- [ ] Ссылка **«обработку персональных данных»** рядом с галочкой открывает страницу `/privacy/`.

### LEAD-02 — Отправка заявки (фронт)
- [ ] Отметить согласие + заполнить контакты → «Отправить заявку» →
      при успехе сообщение **«Спасибо, заявка отправлена. Мы свяжемся с вами в течение часа.»**
      и раскрывается **полная построчная детализация сметы** (воронка D-10).
- [ ] При недоступном сервере → красное сообщение
      **«Не удалось отправить заявку. Попробуйте ещё раз или позвоните нам.»**,
      детализация не раскрывается.

## 4. ⛔ ОБЯЗАТЕЛЬНЫЙ ГЕЙТ: проверка ЖИВОЙ доставки письма (LEAD-02 / D-02)

> Это **приёмочный гейт перед `/gsd-verify-work`**. Выполняется **на заливке**
> (на боевом хостинге Reg.ru), требует **доступов Reg.ru от владельца** (Алексея).
> Без зелёного результата этого раздела этап **не считается сданным**.

### Процедура
- [ ] Залить сайт на хостинг Reg.ru; в `send-lead.php` подставить рабочий домен в константу
      `FROM_EMAIL` (`noreply@<рабочий-домен>`), создать этот ящик-отправитель в панели Reg.ru.
- [ ] На боевом сайте открыть калькулятор → рассчитать → заполнить форму реальными данными →
      отметить согласие → **отправить реальную тестовую заявку**.
- [ ] Проверить почтовый ящик **`MultiCrete@yandex.ru`**: письмо с заявкой **ПРИШЛО**.
- [ ] **Обязательно заглянуть в папку «Спам»** на Яндексе — письмо могло попасть туда.

### Эскалация, если письмо НЕ дошло (по порядку)
- [ ] Проверить, что поле **From** письма — на **собственном рабочем домене**
      (НЕ `@yandex.ru`, иначе Яндекс режет как подделку).
- [ ] Проверить, что используется **порт 587 с TLS** (на Reg.ru порт 25 закрыт),
      и что **SPF** домена настроен/проходит.
- [ ] Проверить, что **ящик-отправитель** (`FROM_EMAIL`) реально существует в панели Reg.ru.
- [ ] Если после этого письмо всё равно не доходит (PHP `mail()` капризничает) —
      **переключиться на запасной приёмник Web3Forms** (D-02):
      на фронте в `calculator/calculator-ui.js` заменить `LEAD_ENDPOINT` на
      `https://api.web3forms.com/submit` и добавить `access_key` в отправляемые данные
      (ключ берётся в личном кабинете web3forms.com; в репозиторий ключ НЕ коммитить).
- [ ] После переключения — **повторить проверку доставки** на `MultiCrete@yandex.ru`
      (снова заглянуть в «Спам»).
