<domain>
## Граница этапа

Этап 1 — ТОЛЬКО движок расчёта на чистом JavaScript (без UI, без сборщика). Движок по входам
(площадь, толщина, класс сложности) выдаёт полную разбивку предварительной стоимости покрытия,
совпадающую с эталонным Excel число-в-число. UI, всплывающее окно, форма заявки — этап 2.

**Входы движка:** площадь детали S (м²), толщина покрытия T (мм), класс сложности (1/2/3).
Флаг «деталь >80 кг» из входов УБРАН (см. решение ниже).

**Выход:** объект с полной разбивкой (трудозатраты построчно, ЕСН, накладные, склад,
вспомогательные материалы, гарантийный резерв, плановые накопления/прибыль, стоимость материалов,
ИТОГО, ₽/м², ₽/(м²·мм)) + список текстовых оговорок.
</domain>

## Зафиксированные решения (LOCKED)

### Эталон и принцип воспроизведения
- **Эталон для движка — `knowledge/калькуляции-исходники/Калькуляция МультиКрит 700.xlsx`** (подтверждён
  владельцем как «обновлённая и верная» версия). Движок обязан совпадать с ним ЧИСЛО-В-ЧИСЛО.
- **Воспроизводим «700» точь-в-точь.** Где более ранний документ `РЕШЕНИЯ-по-калькулятору.md`
  расходится с эталоном — побеждает эталон; расходящиеся правки выносим настраиваемыми параметрами
  «на будущее» (по умолчанию как в «700»), а не применяем сразу.
- Итоговые ячейки эталона: ИТОГО = `G90`/`E34`, ₽/м² = `E36`, ₽/(м²·мм) = `E37`.

### Гарантийный резерв
- **5%** (как в эталоне «700», ячейка E87). Вынести параметром (легко переключить на 3%), умолчание 5%.

### Флаг «деталь тяжелее 80 кг» (меняет CALC-07)
- **НЕ считается отдельной строкой.** Расчёт по умолчанию для детали массой до 80 кг.
- В конце результата — **текстовая оговорка**: «Расчёт сделан для детали массой до 80 кг. При большей
  массе добавится грузоподъёмное оборудование и итоговая стоимость вырастет.»
- Из входов движка флаг исключён.

### Класс сложности (CALC-02)
- Три класса: 1 «простое тело вращения» ×1.0, 2 «сложное тело вращения» ×1.2, 3 «деталь с закрываемыми
  зонами» ×1.4. Коэффициент применяется к ТРУДОЗАТРАТАМ (работам), не к материалам.

### Нанесение по толщине (CALC-03)
- Нелинейно: первые 3 мм по повышенной расценке (900 ₽/(м²·мм)), далее по сниженной (как в «700»).
  Подтверждено: F52 T=2→1800, T=5→3700, излом на 3 мм.

### Материалы (CALC-04)
- Ценовые пороги эластомера по массе: 6800 / 6650 / 6400 ₽/кг.
- **Маржу на материал НЕ добавляем** (производитель, маржа уже в прайсе) — решение владельца.

### Вспомогательные материалы (CALC-05)
- Воспроизводим как в эталоне «700» (там гибрид: часть от массы `E80`, часть от площади `F30`).
  Нормативную правку «строго по площади» из РЕШЕНИЙ оставить параметром на будущее (влияние ~2934 ₽).
- Округление скотча — как в «700» (до 0.1). Правка «до целого» — на будущее.
- Формула итога `=SUM(F3:F18)` в эталоне КОРРЕКТНА («баг сдвига» — ложная тревога, подтверждено).

### Накрутки (CALC-06)
- Порядок: ЕСН на труд → накладные → склад → гарантийный резерв → плановые накопления (прибыль).
- НДС = 0 (УСН).

### Фиксированные допущения (CALC-08)
- По умолчанию: 1 день работ, 1 специалист, без транспорта, без командировок, без спецоснастки.

### Единый блок параметров (CALC-09)
- ВСЕ коэффициенты, ставки, цены, нормы — в одном блоке параметров `PARAMS` (один источник истины),
  не разбросаны по формулам.

### Архитектура
- Чистый JS-модуль `calculator/` по аналогии с существующим `materials/`, без сборщика.
- Ядро: чистая функция `calc(inputs, PARAMS) -> breakdown`.
- Самопроверка на контрольных примерах без тест-фреймворка (прогон в браузере и/или Node).
</decisions>
