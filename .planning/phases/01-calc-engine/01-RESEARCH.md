# Этап 1: Движок расчёта — Исследование

**Дата исследования:** 2026-06-29
**Домен:** Перенос вычислительной модели из эталонного Excel в чистый JS-движок
**Уверенность:** HIGH (модель выгружена из эталона ячейка-в-ячейку и численно сверена — JS воспроизводит ИТОГО до копейки)

## Резюме

Эталонный файл `Калькуляция МультиКрит 700.xlsx` полностью выгружен через PowerShell + Excel COM (три листа: «Расчет. поверхн» 58×12, «Калькуляция» 94×27, «вспомогательные материалы» 18×5) — собраны и формулы (`.Formula`), и значения (`.Value2`). Восстановлена вся цепочка расчёта: геометрия площади → объём и масса эластомера → ценовой порог по массе → построчные трудозатраты × коэффициент класса → ЕСН → вспомогательные материалы → склад → накладные → плановые накопления (прибыль) → основные материалы → гарантийный резерв → ИТОГО без НДС (НДС=0, УСН) → ₽/м² → ₽/(м²·мм).

Модель **уже проверена численно**: JS-реконструкция воспроизводит канонический пример эталона (S=1.6 м², T=5 мм, класс 1) с точностью до копейки — трудозатраты 19 035.24 ₽, материалы 80 792 ₽, ИТОГО 137 600.16 ₽, ₽/м² = 86 000.10, ₽/(м²·мм) = 17 200.02. Это снимает главный риск этапа: метод расчёта однозначен и детерминирован.

Найдено одно расхождение с заданием, которое нужно вынести владельцу: в эталоне «700» **гарантийный резерв = 5 %** (ячейка E87), а в РЕШЕНИЯх владельца и в формулировке требований фигурирует **3 %**. И второе: в эталоне **НЕТ строки «грузоподъёмное оборудование / деталь >80 кг»** (CALC-07) — этого требования в Excel ещё не существует, его нужно спроектировать с нуля (ближайший аналог — строка «Доставка оборудования» G66 = 14 500 ₽ по флагу).

**Главная рекомендация:** реализовать движок как чистую функцию `calc(inputs, PARAMS) -> breakdown` с единым замороженным объектом `PARAMS` (все ставки/цены/нормы — один источник истины, CALC-09) и встроенным мини-набором самопроверок на 3 контрольных примерах из эталона; формулы переносить ровно в том порядке, что в Excel (порядок накруток критичен из-за зависимости «склад от материалов»).

### Зафиксированные решения (выполнять буквально)

- **Три класса сложности 1/2/3** (НЕ A/B/C/D из фоновых файлов). Коэффициент на ТРУДОЗАТРАТЫ (работы), не на материалы:
  - Класс 1 «Простое тело вращения» → ×1.0
  - Класс 2 «Сложное тело вращения (импеллер/винт)» → ×1.2
  - Класс 3 «Деталь с закрываемыми зонами (обклейка/маскировка)» → ×1.4
- **Маржу на материал НЕ добавлять.** МультиКрит — производитель, маржа уже в прайсовой цене. Рекомендацию фонового агента про наценку на материал — игнорировать.
- **Гарантийный резерв ДОБАВИТЬ** отдельной строкой. % на согласовании; в РЕШЕНИЯх и требованиях — 3 %, **в эталоне «700» стоит 5 %** (см. открытый вопрос ОВ-1).
- **«Баг сдвига» вспомогательных материалов — ЛОЖНАЯ ТРЕВОГА, не трогать.** В «700» формула уже корректна: `стоимость = кол-во × цена`, итог `=SUM(F3:F18)`. Подтверждено выгрузкой.
- **Вспомогательные материалы — по нормативам от ПЛОЩАДИ** (не ручной ввод): сопло `ОКРУГЛВВЕРХ(S/10)`, перчатки `S/25`, костюм `S/20`, фильтр `S/25×2×спец`, плёнка `S/50`, скотч `ОКРУГЛВВЕРХ(S/10)`, диск `S/10`. (В эталоне «700» часть нормативов привязана к массе эластомера `E80`, часть к площади `F30` — см. раздел «Вспомогательные материалы» и ОВ-3.)
- **Калькулятор публичный и прозрачный**, считает в браузере. Коэффициенты НЕ прячем. Перенос реальных чисел из приватного Excel в открытый код — намеренное согласованное действие, НЕ утечка.
- **CALC-08 фиксированные допущения:** 1 рабочий день, 1 специалист-полимерщик, без транспорта, без командировок, без спецоснастки.

### Паттерн 1: Чистая функция расчёта
**Что:** `calc(inputs, params)` принимает входы и параметры, возвращает полную разбивку. Без чтения глобалей, без DOM, без даты/случайности — детерминирована.
**Когда:** всегда. Это ядро; UI этапа 2 просто вызовет её и отрисует `breakdown`.
**Пример:**
```javascript
// Источник: реконструкция из Калькуляция МультиКрит 700.xlsx (сверено до копейки)
export function calc(inputs, P) {
  const { S, T, klass, heavy } = inputs;        // площадь, толщина, класс 1/2/3, >80кг
  const coef = klass === 3 ? 1.4 : klass === 2 ? 1.2 : 1.0;   // F20

  // 1) материал
  const volEl  = S * 10000 * T / 10;             // I30, см³
  const massEl = volEl * P.rashodElastomer / 1000 * (1 + P.poteriElastomer); // E33
  const E80    = Math.ceil(massEl);              // ROUNDUP(...,0)
  const cenaEl = E80 < 20 ? P.cenaEl_do20
               : E80 < 100 ? P.cenaEl_20_100 : P.cenaEl_ot100;   // F80
  // ... грунты, очиститель, G86 ...

  // 2) труд
  const F52 = T <= 3 ? T * P.rasc1mm : (T - 3) * P.rascPosled + 3 * P.rasc1mm; // нелинейно
  // ... sumWorks ...
  const G55 = (sumWorks * coef + ITR) * P.esn / 100;
  const G56 = sumWorks * coef + ITR + G55;

  // 3) накрутки (строго в этом порядке)
  const G63 = (G62 + G86) * P.sklad / 100;
  const G64 = G56 * P.nakladnye / 100;
  // ...
  const G75 = (G74 - komandirovka) * P.pribyl / 100;
  const G77 = G74 + G75;

  // 4) итог
  const G87 = (G77 + G86) * P.garantiya / 100;
  const G88 = G77 + G86 + G87;
  const G90 = G88;                               // НДС = 0
  return { /* все строки breakdown + itogo: G90, perM2: G90/S, perM2mm: G90/S/T */ };
}
```

### Паттерн 2: Замороженный блок параметров (один источник истины)
**Что:** все числа эталона в одном `Object.freeze`. Формулы движка ссылаются только на `P.xxx`, никаких «магических чисел» в теле.
**Пример:**
```javascript
// Источник: значения ячеек Калькуляция МультиКрит 700.xlsx
export const PARAMS = Object.freeze({
  rashodElastomer: 1.25,   // кг/(м²·мм), РП C5
  poteriElastomer: 0.10,   // РП D33
  // ... (полный список — в разделе «Полный список параметров»)
});
```

### Паттерн 3: Самопроверка без фреймворка
**Что:** массив `{inputs, expected}` из контрольных примеров эталона + сравнение с допуском (округление). Печатает PASS/FAIL.
**Пример:**
```javascript
// Источник: контрольные примеры из «700»
import { calc } from "./engine.js";
import { PARAMS } from "./params.js";
const near = (a, b, eps = 0.01) => Math.abs(a - b) <= eps;
const cases = [
  { name: "эталон S=1.6 T=5 кл1", in: {S:1.6,T:5,klass:1,heavy:false},
    exp: { itogo: 137600.16, perM2: 86000.10 } },
  // ... класс 2, класс 3 ...
];
let ok = true;
for (const c of cases) {
  const r = calc(c.in, PARAMS);
  const pass = near(r.itogo, c.exp.itogo, 0.02) && near(r.perM2, c.exp.perM2, 0.02);
  console.log(`${pass ? "PASS" : "FAIL"} — ${c.name}: itogo=${r.itogo.toFixed(2)}`);
  ok = ok && pass;
}
if (!ok) process.exitCode = 1;   // в браузере — просто красный лог
```

### Антипаттерны (избегать)
- **Магические числа в формулах** (`*0.302`, `*1.25`) — всё через `PARAMS` (CALC-09). Иначе теряется единый источник истины.
- **Применение коэффициента класса к материалам** — коэффициент только к `sumWorks` (труд). Материалы (G86) не трогать (CALC-02).
- **Округление промежуточных через `toFixed`/Math.round не там, где Excel** — Excel округляет ТОЛЬКО `ROUNDUP` в конкретных ячейках (масса E80, грунты E82/E83/E85, очиститель). Остальное — полный `double`. Лишнее округление сломает совпадение.
- **Перенос карты ячеек из других файлов** (`Калькулятор-МультиКрит.xlsx`, циклон, 2 комплекта) — у «700» своя разметка; смешивать нельзя.

## Полная цепочка расчёта (эталон «700», адреса ячеек)

## Не реализовывать вручную (Don't Hand-Roll)

| Проблема | Не строить | Использовать | Почему |
|----------|-----------|--------------|--------|
| Округление вверх | свой `roundUp` с багами на отриц./плавающей | `Math.ceil(x*10^n)/10^n` | Excel `ROUNDUP(x,n)` = округление ОТ нуля до n знаков; `Math.ceil` совпадает для положительных (все величины ≥0) |
| Совпадение чисел с Excel | decimal/BigNumber | обычный `double` (Number) | Excel сам считает в IEEE-754 double; точная десятичная арифметика РАСХОДИТСЯ с эталоном |
| Хранение параметров | разбросать числа по формулам | один `Object.freeze(PARAMS)` | CALC-09, защита от случайной правки |
| Тестирование | тест-фреймворк + сборщик | `assertEqual` + `console` в одном файле | сайт без сборщика; запуск и в браузере, и в `node` |

**Ключевая мысль:** движок — это перенос арифметики Excel. Любая «улучшенная» точность (decimal) или «правильное» округление сломает совпадение с эталоном. Цель — БИТ-В-БИТ та же арифметика, что в Excel.

## Подводные камни

### Камень 1: ROUNDUP двигает цену скачком
**Что:** масса эластомера округляется вверх до целого (`E80=ROUNDUP(11.0,0)=11`), грунты/очиститель — до 0.1. Округление меняет и стоимость материала, и ценовой порог (20/100 кг), и косвенно (в эталоне) количество расходников, привязанных к `E80`.
**Почему:** `ROUNDUP` — не банковское округление, а всегда ВВЕРХ.
**Как избежать:** `Math.ceil(x * 10**n) / 10**n`. Проверить на границе порога (масса ровно 20.0, 100.0).
**Признак:** расхождение в материалах ровно на «недокруглённый хвост».

### Камень 2: коэффициент класса — только к части труда
**Что:** `F20` умножает `SUM(G42:G53)`, но НЕ умножает ИТР (G54). ЕСН считается уже от `SUM*F20 + G54`.
**Почему:** ИТР — фиксированная дневная оплата, не зависит от сложности нанесения.
**Как избежать:** в коде `(sumWorks*coef + ITR)`, не `(sumWorks + ITR)*coef`.
**Признак:** при классе 2/3 трудозатраты «уезжают» на величину ИТР×(coef−1).

### Камень 3: порядок накруток и «гарантия после прибыли»
**Что:** склад зависит от материалов (G86), которые считаются ПОСЛЕ блока работ; гарантия (G87) применяется к (работы+материалы) уже ПОСЛЕ прибыли.
**Почему:** в Excel это разнесено по листу, порядок неочевиден из вёрстки.
**Как избежать:** считать строго: материал → вспом → склад → накладные → работы(G74) → прибыль(G75) → G77 → гарантия(G87) → G88 → G90.
**Признак:** ИТОГО расходится; гарантия посчитана не от той базы.

### Камень 4: внешняя ссылка на прайс
**Что:** цены эластомера/грунтов в эталоне — формулы `='D:\001\...Прайс 2кв.2024.xlsx'!$E$122`. При открытии без `AskToUpdateLinks=$false` Excel виснет на диалоге.
**Почему:** абсолютный путь к файлу 2 кв. 2024, которого нет на этой машине.
**Как избежать:** при выгрузке — `UpdateLinks=0, ReadOnly=$true` (сделано). В движок берём кэш-значения (6800, 7700, 980 и т.д.) как константы `PARAMS`. Прайс датирован 2 кв. 2024 — пометить дату.
**Признак:** `#ССЫЛКА!` или зависший COM.

### Камень 5: усреднённая толщина и «средняя по сегментам»
**Что:** `H30=AVERAGE(H15:H29)` — простое среднее по заполненным сегментам, не взвешенное по площади. На упрощённом сайте вход T один → среднее = само T, проблемы нет.
**Почему:** эталон рассчитан на многосегментную деталь; сайт берёт один T.
**Как избежать:** на этапе 1 принять T как уже усреднённую величину (один сегмент). Посегментность — v2 (CALC2-01).
**Признак:** не проявляется при одном входе T.

## Открытые вопросы

1. **ОВ-1: Гарантийный резерв — 3 % или 5 %?**
   - Что знаем: эталон «700» E87 = 5 %; РЕШЕНИЯ и требования упоминают 3 % («% на согласовании»).
   - Что неясно: какое число — «обновлённое и верное».
   - Рекомендация: вынести в `PARAMS.garantiya` отдельным числом; по умолчанию взять значение эталона (5 %) и пометить для подтверждения владельцем на этапе планирования/discuss.

2. **ОВ-2: Как устроена строка «деталь >80 кг» (CALC-07)?**
   - Что знаем: в эталоне такой строки НЕТ. Ближайший аналог — «Доставка оборудования» G66 = 14 500 ₽ × флаг (раздел транспорта, ВНЕ базы прибыли через `G74-G68`? нет — G66 входит в G74, значит в прибыль попадает).
   - Что неясно: фиксированная сумма грузоподъёма? её величина? входит ли в базу прибыли и гарантии? отдельная строка или переиспользуем G66?
   - Рекомендация: спросить владельца сумму и место. До ответа реализовать как параметр `PARAMS.gruzopodyemnoe` (по умолчанию, напр., как транспорт — фикс ₽, входит в G74 → в прибыль и гарантию), флаг `heavy` включает строку. Зафиксировать контрольный пример после ответа.

3. **ОВ-3: Вспомогательные — строго по площади или гибрид масса/площадь (как в «700»)?**
   - Что знаем: РЕШЕНИЯ требуют привязки к площади; эталон часть расходников (кисти, ветошь, стаканчики) считает от массы `E80`.
   - Что неясно: сознательный ли это гибрид у владельца или недоведённая правка.
   - Рекомендация: для совпадения с эталоном (CALC-01) переносить как в «700»; пометить для владельца. Влияние на итог мало (G62 ≈ 2934 ₽ из 137.6 тыс.).

4. **ОВ-4: Скотч — округление до целого или до 0.1?**
   - Эталон: `ROUNDUP(F30/10,1)`; РЕШЕНИЯ: `ОКРУГЛВВЕРХ(S/10)` (до целого). Мелочь, но зафиксировать в `PARAMS`/нормативах.
