<!-- Проверенный рецепт сборки — из 03-RESEARCH.md, раздел Code Examples. Исполнитель использует напрямую. -->

Структура шаблона `Входящие/Бланк на опросной лист.docx` (zip, 21 часть). Трогаем ТОЛЬКО `word/document.xml`. Байт-в-байт неизменны: `word/header1.xml`, `word/footer1.xml`, `word/media/image1.jpg`, `word/styles.xml`, `[Content_Types].xml`, все rels и служебные части.

Текущее тело document.xml:
`<w:body><w:p w14:paraId="..." .../><w:sectPr ...>...</w:sectPr></w:body>`
Точка вставки — строго ПЕРЕД `<w:sectPr`. Регулярка замены: `(<w:p [^>]*/>)?<w:sectPr` → `<НАШЕ_ТЕЛО><w:sectPr`.

Доступные стили шаблона (styleId): `1` = heading 1 (заголовки блоков), `2` = heading 2 (подзаголовки 4.1 и т.д.), `ab` = Table Grid (таблицы «Поле|Значение»), `a3` = Strong (жирный инлайн). Шрифт тела — Calibri, размер 11pt, язык ru-RU.

В теле используем ТОЛЬКО префикс `w:` (уже объявлен в корне). Свои xmlns НЕ добавлять. `w14:paraId` в наших параграфах опускаем.

Чекбокс = символ `☐` U+2610 прямо в `<w:t>` (Calibri содержит глиф). НЕ использовать `w:sym`/Wingdings.

Экранирование текста между `<w:t>…</w:t>`: `&`→`&amp;` (первым), `<`→`&lt;`, `>`→`&gt;`. Кириллица в UTF-8 экранирования не требует.

`xml:space="preserve"` обязателен в `<w:t>`, где есть значимые ведущие/множественные пробелы (прочерки, подчёркивания).

После каждой `</w:tbl>` — пустой `<w:p/>` (Word не любит две таблицы подряд).
