# Phase 2 — UI Design Contract

> Визуальный и интеракционный контракт окна калькулятора, формы заявки и страницы политики.
> Сгенерирован gsd-ui-researcher, проверяется gsd-ui-checker.
>
> **Источник истины токенов:** `landing.html` `:root` (тёмная тема «Стальной Vega»).
> CONTEXT (D-08, раздел «Бренд») требует, чтобы калькулятор совпадал с тёмной темой
> `landing.html` (`--blue`/`--pink`/`--steel`), а НЕ со светлой палитрой `assets/styles/base.css`.
> Калькулятор переиспользует готовые классы лендинга: `.modal`/`.modal-overlay`/`.modal-close`,
> `.btn`/`.btn-primary`/`.btn-secondary`, `.pf-input`/`.pf-label`/`.pf-row`/`.pf-consent`/`.pf-submit`.

## Spacing Scale

8-point база (DESIGN.md: «базовая единица шага 8 px, всё кратно»). Значения в rem при 16px root.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (.25rem) | Зазор иконка-текст, мелкие inline-отступы |
| sm | 8px (.5rem) | Компактные зазоры, gap внутри строки результата |
| md | 16px (1rem) | Базовый отступ полей, gap в `.pf-row`, padding строк «шапки» |
| lg | 24px (1.5rem) | Внутренние отступы блоков, `.wrap` padding |
| xl | 32px (2rem) | Padding модального окна `.modal` (`2rem`), разделение блоков (поля / результат / форма) |
| 2xl | 48px (3rem) | Крупные разрывы между смысловыми зонами окна |
| 3xl | 64px (4rem) | Резерв для страницы политики (отступы секций) |

**Exceptions:**
- Цель касания (touch target) кнопок/чекбокса на телефоне — минимум 44px высоты (WCAG/iOS). `.btn` уже даёт `padding .85rem 1.75rem` → ≥44px.
- `.modal` на телефоне (`max-width: 560px`): `padding: 0`, окно во весь экран `100dvh` — спец-случай адаптива (RESEARCH «Скользкий участок 3»), а не нарушение шкалы.

## Typography

Точно 4 размерных роли, 2 веса (400 Regular / 600 SemiBold; для CTA-кнопок исключение 700 — наследуется от `.btn`, см. ниже). Числа результата — `font-variant-numeric: tabular-nums`.

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Display (заголовок окна / ИТОГО) | 28px (1.75rem) | 600 | 1.15 | Oswald (`--f-head`) |
| Heading (подзаголовки блоков: «Результат», «Расшифровка классов», «Оставить заявку») | 20px (1.25rem) | 600 | 1.2 | Oswald (`--f-head`) |
| Body (значения «шапки», оговорки, текст) | 16px (1rem) | 400 | 1.5 | Manrope (`--f-body`) |
| Label (подписи полей, подписи строк сметы, метка раздела) | 13px (.8rem) | 600 | 1.3 | Manrope / Tektur для uppercase-меток |

**Веса:** Regular 400 (текст/значения) + SemiBold 600 (заголовки/подписи) — это два авторских веса контракта. **Исключение:** вес 700 присутствует ТОЛЬКО в тексте кнопок `.btn`/`.pf-submit` — он унаследован из существующего CSS лендинга (`landing.html` строка 107), НЕ является авторским токеном контракта и изменению не подлежит (D-08). При подсчёте весов контракта 700 не учитывается.

**Откуда:** `landing.html` строки 36-38, 53, 70 (clamp заголовков), 107, 329; DESIGN.md (Oswald 20-24 / Manrope 14-16 / табличные цифры). ИТОГО — крупно (Display), это смысловой акцент результата (UI-03).

**Главный визуальный якорь (focal point):** блок ИТОГО — Display 28px, цвет `--blue`, выровнен так, чтобы первым притягивать взгляд при появлении результата. Иерархия внимания: ИТОГО (Display/синий) → подзаголовки блоков (Heading) → значения «шапки» и текст (Body) → подписи полей (Label). Исполнитель не должен выводить приоритет догадками — главный якорь объявлен здесь явно.

## Color

Палитра «Стальной Vega» (тёмная тема). 60/30/10 на фоне тёмного окна.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#212C3D` (`--steel`) | Фон модального окна `.modal`, базовая поверхность |
| Secondary (30%) | `#19222F` (`--scale`) / `#1A2332` (`--bg`) | Утопленные шкалы, фон полей ввода `.pf-input`, оверлей-затемнение фона |
| Accent (10%) | `#EB4D8C` (`--pink`, нижняя грань `#A52350` `--pink-d`) | СТРОГО перечень ниже |
| Info (синий) | `#2F66A5` (`--blue`) | Информационный акцент: ИТОГО-число, focus-обводка полей, числа в «шапке», вторичная кнопка-контур |
| Destructive | `#D64550` (`--red`) | ТОЛЬКО текст ошибок валидации и ошибки отправки (единственное исключение из палитры — DESIGN.md) |

**Accent (розовый) reserved for — точный список (никогда «все интерактивные»):**
1. Главная CTA-кнопка вызова калькулятора на лендинге (`.btn-primary`, INT-02).
2. Кнопка «Рассчитать» внутри окна (главное действие расчёта).
3. Кнопка «Отправить заявку» / `.pf-submit` (главное действие формы, LEAD-01).
4. Тонкая светящаяся линия-акцент под блоком ИТОГО (опц., приём `card-hero::after` из DESIGN.md).

**Разделение цветов (правило бренда):** синий `--blue` = информация (число ИТОГО, focus, подписи-данные); розовый `--pink` = действие (только кнопки выше). Не смешивать: ИТОГО-сумма — синяя (это информация), кнопка под ней — розовая (это действие).

**Контраст:** розовый текст на тёмном/белом НЕ использовать (DESIGN.md: 3.1:1) — розовый только как заливка кнопок с белым текстом. Светлый текст `--ink #EAF0F7` на `--steel #212C3D` — высокий контраст, ОК.

**Откуда:** `landing.html` строки 17-33, 113-124, 330-333, 358/410; DESIGN.md (разделение цветов, красный-исключение, контраст).

## Interaction Contract (этап-специфично)

| Элемент | Контракт |
|---------|----------|
| Открытие окна | Клик по `.btn-primary` «Получить расчёт» → `.modal-overlay.open`, `body{overflow:hidden}` (UI-01, переиспользуем `openModal`-механику). |
| Закрытие окна | Крестик `.modal-close`, клик по фону (`handleOverlay`), `Esc` — все три (UI-01). |
| Фокус | При открытии — фокус на первое поле; при закрытии — возврат на кнопку-вызов. `role="dialog"`, `aria-modal="true"`, `aria-labelledby`. |
| Поля ввода | Площадь (`number`, м²), толщина (`number`, мм), класс (radio/select 1·2·3), чекбокс «деталь тяжелее 80 кг» (UI-02). |
| Чекбокс «>80 кг» | НЕ влияет на число — только меняет текст оговорки (RESEARCH «Контракт движка»; CALC-07). |
| Валидация | Положительные числа + класс выбран; ошибка → подсказка красным `--red` под полем, расчёт не вызывается (UI-05, D-12). Момент валидации (inline/при сабмите) — на усмотрение исполнителя. |
| Результат «шапка» (СРАЗУ) | Строки «подпись — значение» (`tabular-nums`, выравнивание значений по правому краю); ИТОГО — крупно, синим. Поля из `breakdown`: trudozatraty, vspomMaterialy, nakladnye, pribyl, nds, transport/komandirovki=0, materialy, itogo, perM2 (UI-03, D-09). |
| Детализация (за контактами) | Скрытый блок `hidden`; раскрывается после валидной отправки формы (D-10). Это раскладка, не цена → клиентский show/hide, без серверной защиты. |
| Адаптив | `max-height: 90dvh` + `overflow-y:auto` + `overscroll-behavior:contain`; на `≤560px` — `100dvh`, `border-radius:0`, `env(safe-area-inset-bottom)`. `.pf-row` → 1 столбец на `≤768px` (UI-06, RESEARCH «Скользкий участок 3»). |
| Анимация | Без агрессивных эффектов; уважать `prefers-reduced-motion` (DESIGN.md). |
