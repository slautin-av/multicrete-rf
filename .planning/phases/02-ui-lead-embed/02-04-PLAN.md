## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| браузер (форма) → send-lead.php | Поля заявки уходят POST'ом на обработчик (серверная защита — план 02). |
| клиент → блок детализации | Раскрытие #calc-details — клиентский show/hide. Это раскладка (не цена, не секрет) → серверная защита не нужна (RESEARCH «Responsibility Map»). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Repudiation / Compliance | Отправка без согласия 152-ФЗ | mitigate | Чекбокс согласия required → form.checkValidity() блокирует отправку без него; ссылка ведёт на /privacy/. Согласие активное (без checked). |
| T-02-12 | Information Disclosure | «Детализация за контактами» в DOM | accept | Детализация — раскладка стоимости, НЕ секрет и НЕ цена (цена видна сразу). Клиентский show/hide достаточен; серверная защита избыточна (осознанное решение D-10). |
| T-02-13 | Spoofing | Бот заполняет форму | mitigate | Honeypot website отправляется на сервер, где тихо отбрасывается (план 02). |
| T-02-14 | Information Disclosure | Web3Forms access_key (если fallback) | mitigate | Ключ Web3Forms публичен по дизайну, но в этот этап НЕ зашиваем (только /send-lead.php). При включении fallback ключ — в форму/конфиг, не плодить в репозитории (CLAUDE.md). |
</threat_model>
