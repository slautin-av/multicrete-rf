# ARCHITECTURE — Архитектура

## Тип приложения

Статический многостраничный сайт. Без сборщика, без фреймворка, без бэкенда. Все страницы — отдельные `.html`-файлы, открываемые напрямую в браузере или через любой статический хостинг. Деплой — прямая отдача файлов (например, Nginx, GitHub Pages, любой CDN).

## Точки входа

| Файл | Роль |
|---|---|
| `landing.html` | **Основная лендинговая страница** (1004 строки). Полностью самодостаточная: содержит весь CSS в `<style>`, все скрипты в `<script>`. Навигация ведёт к якорям (`#spheres`, `#portfolio`, `#specs`, `#price`, `#contacts`) внутри той же страницы. |
| `index.html` | Запасная точка входа (265 строк). Подключает `assets/styles/base.css`, имеет полноценный хедер с навигацией по разделам сайта (`materials/`, `cases/`, `knowledge/`, `about/`). Содержит блок ДО/ПОСЛЕ с эластомерной бронёй. Технически — «старая» версия главной, параллельная `landing.html`. |
| `materials/index.html` | Каталог материалов (355 строк). Подключает `../assets/styles/base.css` и `../assets/styles/catalog.css`. |
| `do-posle-demo.html` | Изолированная демо-страница блока «ДО / ПОСЛЕ» (172 строки). Не часть навигации. |
| `smoke-demo.html` | Изолированная демо-страница WebGL-дыма (137 строк). Не часть навигации. |

Связь между страницами: ссылки в хедере `index.html` и `materials/index.html` ведут на `about/`, `cases/`, `knowledge/`, `problems/`, `contact/` — но эти папки **пусты** (файлов `.html` внутри нет). Разделы существуют как концепция, но страницы ещё не созданы.

## Слои / разделение

```
Разметка:  *.html — содержит структуру страницы + контент
Стили:     assets/styles/base.css     — общие токены и компоненты (хедер, футер, кнопки, сетка)
           assets/styles/catalog.css  — стили каталога материалов
           landing.html <style>       — весь CSS лендинга встроен в файл (inline, ~470 строк CSS)
           index.html <style>         — часть стилей встроена (только специфичные для страницы)
Скрипты:   assets/js/webgl-fluid.umd.js — сторонняя библиотека WebGL-симуляции жидкости
           landing.html <script>      — вся JS-логика встроена: модальное окно, табы, якорная навигация,
                                        зум ДО/ПОСЛЕ, проброс mousemove в WebGL-холст
           https://unpkg.com/lucide@latest — иконки (CDN, внешняя зависимость)
Шрифты:    Google Fonts CDN — Oswald, Manrope, Tektur (внешняя зависимость)
```

Нет единой JS-точки входа (нет `main.js`). Нет модульной системы. Каждая страница — монолит или подключает общий base.css.

## Поток данных

Полностью статичный сайт. Никаких запросов к API, никакого состояния на сервере.

Есть JS-состояние **на клиенте**, локальное для `landing.html`:
- Переключение вкладок в блоке ТТХ (`switchTab()`)
- Открытие/закрытие модального окна (`openModal()` / `closeModal()`)
- Зум изображений ДО/ПОСЛЕ по клику (toggles CSS-класс `is-zoomed`)
- Якорная прокрутка с плавным scroll в nav-ссылках
- WebGL-симуляция дыма: реагирует на `mousemove`, `mousedown`, `touchmove` — данные не сохраняются

Форма обратного звонка в `landing.html` имеет HTML-разметку (`<form class="pf-form">`), но обработчик отправки в коде не реализован — форма декоративная / заглушка.

## Переиспользуемые блоки

Паттерны, повторяющиеся на нескольких страницах:

1. **Хедер сайта** — реализован по-разному в `landing.html` (`.header > .nav`, тёмная тема, встроенный CSS) и в `index.html` / `materials/index.html` (`.site-header > .nav-inner`, подключает `base.css`). Единого компонента нет — дублирование разметки.

2. **Футер** — аналогично: разная разметка в `landing.html` (`.footer`) и `index.html` / `materials/index.html` (`.site-footer`).

3. **Палитра бренда** — CSS-переменные (`:root`) объявлены в `base.css` (светлая тема) и **переопределены заново** в `landing.html <style>` (тёмная тема «Стальной Vega»). Переменные не синхронизированы: `base.css` использует `--color-primary`, `landing.html` использует `--blue`, `--pink`, `--steel` и т.д.

4. **Карточка (`.card`)** — скевоморфный компонент с тенью и рельефом. Определён в `landing.html`, отсутствует в `base.css`.

5. **Блок ДО / ПОСЛЕ** — реализован в `index.html` как `.mc-armor`, и вынесен отдельно в `do-posle-demo.html` для тестирования. В `landing.html` есть визуально похожий блок `.dp__pair`.

6. **Гексагональные иконки** — `.hex { <i data-lucide="..."> }` — паттерн декоративного оформления иконок, используется по всей `landing.html`.

## Как встроить новое

**Калькулятор расхода/стоимости:**
Логично разместить как отдельную страницу `calculator/index.html` (по аналогии с `materials/index.html`). Подключает `../assets/styles/base.css` плюс собственный `../assets/styles/calculator.css`. Вся логика — встроенный `<script>`. Ссылку добавить в футер и в навигацию хедера. Альтернатива — встроить секцией в `landing.html` как `<section id="calculator">`.

**Каталог кейсов:**
Папка `cases/` уже зарезервирована и фигурирует в навигации `index.html` и `materials/index.html`. Нужно создать `cases/index.html` (список объектов) и, опционально, `cases/[slug]/index.html` для отдельных кейсов. Стили — добавить `assets/styles/cases.css` по образцу `catalog.css`. Изображения кейсов хранить в `assets/images/cases/`.
