# STACK — Технологии

## Языки и среда выполнения

Чистый стек без сборщика, без npm, без Node.js:
- **HTML5** — разметка всех страниц
- **CSS3** — стили (внешние файлы + инлайн-блоки внутри `<style>` в HTML)
- **Vanilla JavaScript (ES6+)** — вся логика на клиенте, без фреймворков

Деплой/разработка — статические файлы, открываются напрямую в браузере.

## Фреймворки и библиотеки

| Библиотека | Версия / источник | Что делает |
|---|---|---|
| **WebGL Fluid** | `assets/js/webgl-fluid.umd.js` (локальный UMD-бандл) | Интерактивный эффект дыма на `<canvas>` с реакцией на курсор/тач |
| **Lucide Icons** | `https://unpkg.com/lucide@latest` (CDN, latest) | SVG-иконки; подключаются через `<script>`, активируются вызовом `lucide.createIcons()` |

Внешние CSS-фреймворки (Bootstrap, Tailwind и т.п.) — **не используются**.

## Стили

### Файловая структура CSS

- `assets/styles/base.css` — общий сброс, CSS-переменные палитры «старой» светлой темы, типографика, шапка, кнопки. Подключается в `index.html` и `materials/index.html`.
- `assets/styles/catalog.css` — специфические стили каталога материалов. Подключается в `materials/index.html`.
- **Инлайн-стили в `landing.html`** — весь CSS лендинга вынесен в большой блок `<style>` внутри `<head>`. Это самый актуальный и детально проработанный файл стилей (тёмная тема, дым, скевоморфизм).
- **Инлайн-стили в `smoke-demo.html`** и **`do-posle-demo.html`** — изолированные стили демо-страниц.

«Стальной Vega + розовый неон» — скевоморфизм: карточки с физическим объёмом (box-shadow), утопленные шкалы (inset shadows), кнопки-клавиши с нижней гранью. Описание техник — в `DESIGN.md`.

## JavaScript

### Инлайн-скрипты в `landing.html`

Вся логика — в `<script>`-блоке перед `</body>`:

- `lucide.createIcons()` — инициализация иконок Lucide
- `openModal()` / `closeModal()` / `handleOverlay()` — управление модальным окном «Заказать звонок»
- `goTo(s)` — плавный скролл к секции
- `switchTab(t)` — переключение вкладок таблицы ТТХ (материал A / материал B)
- `submitForm(e)` — обработка формы «Получить прайс» (только `alert`, реального API-запроса нет)
- `submitCall(e)` — обработка формы звонка (только `alert`)
- Клик по якорным ссылкам — плавный скролл через `scrollIntoView`
- Zoom картинок ДО/ПОСЛЕ по клику (класс `is-zoomed`)
- Инициализация `WebGLFluid` на `<canvas id="fluid-bg">` с параметрами цветовой палитры
- Проброс событий мыши/тача с `window` на `fluidCanvas` (дым реагирует на курсор везде)

### Внешние JS-файлы

- `assets/js/webgl-fluid.umd.js` — WebGL-симуляция жидкости (self-contained UMD-бандл, ~1 файл)

### Прочие страницы

- `materials/index.html` — JS отсутствует (только Lucide CDN, если подключён; в просмотренном коде нет)
- `smoke-demo.html`, `do-posle-demo.html` — изолированные демо с собственными inline-скриптами

## Конфигурация и окружение

Файл `.env` **присутствует** в корне проекта (добавлен в `.gitignore`, значения не читались).

По `.env.example` ожидаются следующие переменные:

| Переменная | Назначение |
|---|---|
| `ANTHROPIC_API_KEY` | Ключ Claude API (Anthropic) |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота для уведомлений с сайта |
| `TELEGRAM_CHAT_ID` | ID чата Telegram для приёма уведомлений |
| `CONTACT_EMAIL` | Контактный email для форм обратной связи (если появится backend) |

На текущем этапе бэкенда нет — переменные зарезервированы на будущее.

## Хостинг / деплой

В `README.md` и конфигах информация о хостинге **не обнаружена**. Нет CI/CD конфигов (`.github/`, `netlify.toml`, `vercel.json` и т.п.).
Домен `www.MultiCrete.ru` упоминается в контактах `landing.html`, но конфигурации деплоя нет.
