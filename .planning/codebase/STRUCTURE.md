## Где код, а где материалы/заметки

**Рабочий код сайта:**
- `landing.html`, `index.html`, `materials/index.html` — страницы сайта
- `do-posle-demo.html`, `smoke-demo.html` — рабочие демо (разработка UI)
- `assets/styles/`, `assets/js/` — стили и скрипты
- `assets/images/` — изображения, используемые в `<img src="...">` на страницах

**Материалы, заметки, референсы (не код):**
- `_research/` — дизайн-исследования, кандидаты компонентов, референсы палитры
- `Картинки/` — исходники фото для будущей загрузки; документы оборудования (Word, DOCX)
- `памятки/` — личные заметки по проекту
- `presentation/` — PDF-документы для клиентов, не публикуются на сайте
- `ВИДЕНИЕ-ПОРТАЛА.md` — стратегия/видение, не код
- `knowledge/`, `about/`, `cases/`, `problems/` — **зарезервированные папки**, содержимое не создано

## Ассеты

**`assets/`** — единственная официальная папка ассетов для кода:
- `assets/styles/base.css` — глобальная дизайн-система (светлая тема; переменные `--color-*`)
- `assets/styles/catalog.css` — специфичные стили страницы каталога
- `assets/js/webgl-fluid.umd.js` — WebGL-библиотека (~минифицированный бандл)
- `assets/images/` — 3 изображения в корне + папка `sections/` с 3 изображениями для лендинга

Внимание: `landing.html` определяет **свою тёмную палитру** прямо в `<style>` через переменные `--blue`, `--pink`, `--steel`, `--ink` и т.д. — они **не совпадают** с переменными `base.css` (`--color-primary`, `--color-dark`). Это два разных дизайн-языка в одном проекте.

**`Картинки/`** — неструктурированное хранилище исходников. Часть изображений уже переименована и перемещена в `assets/images/` (например, `Logo_Multicrete.png`). Папка `Архив/` внутри содержит старые версии.

## Соглашения именования файлов

По правилам проекта (`CLAUDE.md`):
- Файлы **кода** — латиница и дефисы: `landing.html`, `base.css`, `webgl-fluid.umd.js`, `do-posle-demo.html`
- Папки заметок/материалов — кириллица: `Картинки/`, `памятки/`, `ВИДЕНИЕ-ПОРТАЛА.md`
- Нарушения: `Картинки/` содержит `.docx`-файлы с кириллицей в именах; файл `ВИДЕНИЕ-ПОРТАЛА.md` в корне репозитория

## Ключевые места

В первую очередь смотреть сюда:

1. `landing.html` — основная продуктовая страница, вся дизайн-система тёмной темы встроена в `<style>` в head
2. `assets/styles/base.css` — общие токены и компоненты для многостраничной части сайта
3. `assets/styles/catalog.css` — стили каталога
4. `materials/index.html` — единственный полностью готовый раздел, кроме лендинга
5. `DESIGN.md` — дизайн-контракт: концепция «Стальной Vega», скевоморфизм, ссылка на `brand.md`
6. `_research/design-refs/` — история дизайн-решений и кандидаты компонентов
