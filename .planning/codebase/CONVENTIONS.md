# CONVENTIONS — Соглашения по коду

## Общий стиль вёрстки

**Семантика.** Используются семантические теги HTML5: `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`. В `landing.html` структура полностью инлайновая (один файл), в `index.html` и `materials/index.html` — подключаются внешние CSS-файлы (`assets/styles/base.css`, `assets/styles/catalog.css`).

**Инлайн vs внешние стили.** Два подхода сосуществуют:
- `landing.html` — весь CSS написан в `<style>` прямо в `<head>` (один автономный файл, несколько сотен строк CSS). Это сознательное решение для лендинга с тёмной темой и дымом — изолированный «экземпляр» дизайна.
- `index.html`, `materials/index.html` — подключают внешние файлы `base.css` + `catalog.css`; минимальные инлайн-стили только для точечных переопределений.

**Отступы.** В HTML — 2 пробела. В CSS — 2 пробела. Строки не ограничены жёстко, но правила чаще умещаются в 1–3 строки, при необходимости — многострочные `box-shadow`.

**Изображения.** `max-width: 100%; display: block;` по умолчанию. Ленивая загрузка (`loading="lazy"`) в demo-файлах не проставлена, в основных страницах — не замечена.

**Скрипты.** Подключаются в конце `<body>`. Lucide-иконки грузятся с CDN (`unpkg.com/lucide`). WebGL-дым — локальный файл `assets/js/webgl-fluid.umd.js`.

## CSS: именование классов и методология

**Методология.** Строгого BEM нет. Применяется смешанный подход:

- В `landing.html` — плоские утилитарные классы (`.wrap`, `.section`, `.card`, `.btn`, `.btn-primary`, `.btn-secondary`, `.hex`, `.sec-label`, `.sec-title`, `.grid-3`, `.grid-4`). Модификаторы через второй класс (`.tech-card.bad`, `.tech-card.good`).
- В `catalog.css` — частичный BEM: блок + элемент через двойное подчёркивание (`.catalog-hero__inner`, `.catalog-hero__usp`, `.catalog-hero__code-legend`), модификаторы через двойное тире (`.series-block--concrete`, `.series-block--metal`, `.task-badge--concrete`, `.task-badge--emergency`).
- В `base.css` — плоские классы навигации и футера (`.site-header`, `.nav-inner`, `.nav-logo`, `.nav-links`, `.nav-cta`, `.footer-inner`, `.footer-brand`, `.footer-col`, `.footer-bottom`).

**Соглашение по именам классов.** Строчные буквы, слова разделяются дефисом (`-`). Кириллица в именах классов не используется.

**Структура секций в `landing.html`.** Каждая секция имеет префикс по смыслу: `.hero-*`, `.problem-*`, `.tech-*`, `.roi-*`, `.cmp-*`, `.specs-*`, `.sphere-*`, `.app-*`, `.contact-*`. Компонентные классы без префикса: `.wrap`, `.section`, `.card`, `.btn`, `.hex`, `.spec-cell`.

### Переменные в `base.css` (светлая тема, ранний слой)

```css
:root {
  --color-primary:  #C8192A;   /* старый красный — устаревший, не совпадает с брендом */
  --color-accent:   #1A4F8A;
  --color-dark:     #1E1E2E;
  --color-surface:  #F5F6F7;
  --color-text:     #1E1E1E;
```

> Внимание: `base.css` содержит устаревшую палитру (`--color-primary: #C8192A` — красный), не совпадающую с утверждённым брендом (`--pink: #EB4D8C`). Этот файл относится к раннему итерации сайта; `landing.html` использует собственный `:root` и `base.css` не подключает.

## JavaScript-стиль

Единственный проектный JS — в конце `landing.html` в теге `<script>` (инлайн). Стиль:
- Обычный ES6+, без фреймворков, без сборки.
- Именование функций — camelCase на латинице: `openModal`, `closeModal`, `handleOverlay`, `goTo`, `switchTab`, `submitForm`, `submitCall`.
- Короткие однострочные функции, без классов.
- Комментарии — на русском языке, поясняют поведение.

Сторонние скрипты:
- `assets/js/webgl-fluid.umd.js` — минифицированный UMD-бандл сторонней библиотеки WebGL-симуляции жидкости.
- Lucide подключается с CDN (`unpkg.com/lucide@latest`), инициализируется через `lucide.createIcons()`.
